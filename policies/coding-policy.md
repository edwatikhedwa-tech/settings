# Coding Policy

- Изменения должны быть узкими и проверяемыми.
- Скрипты установки обязаны поддерживать `-Plan`, `-DryRun` и явный `-Apply`.
- Повторный запуск не должен ломать уже установленную конфигурацию.
- Для JSON, YAML и TOML используй структурные парсеры или ограниченные управляемые блоки.
- Не заменяй пользовательский файл целиком, если можно изменить только управляемую секцию.
- Перед изменением пользовательского файла делай резервную копию.
